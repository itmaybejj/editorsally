## Project Overview

This is the marketing and documentation website for **Editoria11y**, an open-source accessibility checker tool. The site covers Drupal/WordPress integrations, installation guides, features, and community membership. Domain: editoria11y.com.

## Local Development

This project uses **DDEV** for local development (Docker-based):

```bash
ddev start          # Start the local environment
ddev stop           # Stop the environment
ddev describe       # Show project info (local URL, etc.)
```

The site is served from the project root (no separate docroot) via nginx-fpm.

To update the bundled Editoria11y library from GitHub:
```bash
bash scripts/get.sh
```

## Architecture

**Pure static HTML** — no build pipeline, no package manager. Pages are hand-authored HTML files under `en/<topic>/index.html`. The "about" page lives at `en/index.html` (language root), not `en/about/`.

- **Styling**: Bootstrap 5 (`assets/bootstrap/`) + custom overrides in `assets/theme.css`
- **JavaScript**: `assets/theme.js` handles nav population, language routing, active link highlighting, language picker, and responsive offcanvas menu. `assets/lang/i18n.js` provides i18n configuration (path mappings, nav strings, native language names, supported languages) consumed by theme.js. `assets/demo.js` powers the live accessibility checker demos.
- **Syntax highlighting**: Prism.js (`assets/prism/`)
- **Editoria11y library**: Bundled at `assets/ed11y/` (pulled via `scripts/get.sh` from the `3.0.x-dev` branch)
- **Freemius**: Payment/plugin management CSS in `assets/freemius/`

## Page Structure

All content pages follow the structure in `template.html`. Pages live at `en/<topic>/index.html` (except "about" which is at `en/index.html`). Each page's `<header>` contains a `<div id="lang-picker"></div>` placeholder that `theme.js` populates with a language-switching dropdown. Navigation links are dynamically injected by `theme.js` using strings from `assets/lang/i18n.js`.

Language-detection stub pages at the site root (`/features/index.html`, `/license/index.html`, etc.) redirect visitors to the appropriate language version based on browser preferences. The root `/index.html` redirects to `/{lang}/`.

Light/dark mode is controlled via the `data-bs-theme` attribute on `<html>`. The primary brand color is `#712cf9` (purple).

## Translations

Translated pages live at `<langCode>/<topic>/index.html` (e.g., `es/features/index.html`). The "about" page for each language lives at `<langCode>/index.html` (e.g., `es/index.html`). The English `en/` pages are the canonical source.

- **Configuration**: `assets/lang/i18n.js` — path mappings, nav strings per language, supported language list
- **Staleness tracking**: `lang/manifest.json` — records which English git commit each translation was last synced to. Use `git diff <hash> HEAD -- en/<page>/index.html` to detect stale translations.
- **Skill**: Use `/translations <langCode> [page]` to translate or update pages. Use `/translations --status` for a staleness report.
- **Sitemap**: `sitemap.xml` is generated by `node scripts/generate-sitemap.js`. It walks `i18n.canonicalPaths` × `i18n.allLanguages` and emits a `<url>` entry with `xhtml:link rel="alternate"` hreflang pointers for every page that exists on disk. Regenerate after adding, moving, or deleting a page or language.

To add a new language:
1. Add nav strings and (optional) path mappings in `assets/lang/i18n.js`
2. Translate pages using the `/translations` skill
3. Uncomment the language code in `i18n.js` `supportedLanguages` when ready for visitors
