Funcionalidades

Os testes principais do Editoria11y detectam mais de 50 problemas comuns de acessibilidade de conteúdo: decisões que prejudicam a legibilidade, erros que tornam os leitores de tela menos eficazes e falhas que reduzem o SEO.

Os complementos oferecem aproximadamente 40 testes adicionais para desenvolvedores e designers, além de testes personalizados e diversas ferramentas de garantia de qualidade em todo o site.

Os testes de conteúdo focam em problemas que surgem após os sites serem entregues aos autores de conteúdo. Os sites precisam ser testados com ferramentas de auditoria manual, teclados e leitores de tela antes do lançamento, mas mesmo o site mais acessível no primeiro dia tende a se deteriorar à medida que edições introduzem novos problemas.

O que faz o Editoria11y se destacar entre as ferramentas de acessibilidade é que seus testes rodam ao vivo, no navegador, enquanto os autores editam e visualizam seu trabalho. Os autores reagem com muito mais facilidade a um feedback imediato e inline, percebem os alertas e fazem correções, do que se lembram de visitar um painel ou executar uma verificação manual.

A experiência do autor

Os autores veem a barra de ferramentas do Editoria11y enquanto editam o conteúdo. Quando problemas são detectados, ela fica amarela ou vermelha e exibe o número de ocorrências. Clicar no contador rola a página até o primeiro alerta e abre seu tooltip.

As dicas aparecem inline, com uma descrição do erro em linguagem simples, sugestões de correção e notas sobre por que o problema é importante.

A barra de ferramentas contém recursos adicionais para verificações manuais, incluindo visualizadores de estrutura de títulos e texto alternativo, além de pontuação de legibilidade.

Controles do administrador do site

Todos os aspectos visíveis do verificador podem ser personalizados. Parâmetros e eventos permitem remover testes, reescrever dicas, inserir testes personalizados e definir temas customizados.

Os plugins para Drupal e WordPress adicionam recursos extras, como dispensa sincronizada de alertas, painéis de todo o site e exportações de relatórios em CSV.

Testes

Os testes do Editoria11y focam em melhorar a experiência da Web para todos os usuários, medida por seres humanos, e não apenas por padrões técnicos. Embora os testes principais visem ajudar a alcançar as Diretrizes de Acessibilidade para Conteúdo Web (WCAG 2.2 AA), outros são influenciados pelo trabalho da Força-Tarefa de Acessibilidade para Deficiências Cognitivas e de Aprendizagem, bem como pelas melhores práticas gerais de usabilidade e design.

Testes individuais podem ser desativados pelos administradores do site.

Texto alternativo de imagens

  • Imagem sem atributo alt
  • Texto alternativo é um nome de arquivo
  • Texto alternativo é um marcador de posição ("A definir")
  • Texto alternativo contém apenas símbolos impronunciáveis
  • Texto alternativo contém palavras redundantes ("imagem de…")
  • Texto alternativo é longo demais
  • Imagem decorativa pode ter valor informativo
  • Imagem de carrossel marcada como decorativa
  • Imagem com legenda não tem texto alternativo
  • Texto alternativo duplica a legenda

Imagens com link

  • Imagem com link não tem texto alternativo
  • Texto alternativo da imagem com link é uma URL ou nome de arquivo
  • Texto alternativo da imagem com link é um marcador de posição
  • Texto alternativo da imagem com link contém apenas símbolos impronunciáveis
  • Texto alternativo descreve a imagem, não o link
  • Texto alternativo da imagem com link contém palavras redundantes
  • Texto alternativo da imagem com link é longo demais
  • Imagem em link com texto não tem atributo alt

Mídia incorporada

  • Vídeo precisa de legendas
  • Áudio precisa de transcrição
  • Visualização de dados precisa de alternativa acessível
  • Frame sem título
  • Frame excluído da navegação por teclado
  • Conteúdo do frame precisa de verificação manual de acessibilidade
  • Documento vinculado pode não ser acessível a leitores de tela
  • PDF não possui alternativa acessível
  • Componentes de layout interativo aninhados

Links significativos

  • Link vazio
  • Texto do link é uma URL
  • Texto do link é apenas um número DOI
  • Texto do link diz "clique aqui"
  • Texto do link contém apenas palavras genéricas ("leia mais")
  • Links diferentes compartilham o mesmo texto
  • Ícone ou imagem com link não tem alternativa em texto
  • Link abre nova aba sem aviso
  • Link aponta para um arquivo sem aviso
  • Texto do link contém apenas símbolos ou emoji
  • Texto significativo do link oculto para usuários com visão
  • Tooltip de link redundante
  • Link âncora interno quebrado
  • Atributo ID duplicado
  • Link pode apontar para um ambiente de desenvolvimento

Títulos

  • Página não tem Título 1
  • Primeiro título é um subtítulo
  • Título pula um nível
  • Título está vazio
  • Imagem usada como título precisa de texto alternativo
  • Título é muito longo
  • Parágrafo em negrito pode ser um título
  • Bloco de citação curto pode ser um título

Legibilidade do texto

  • Texto em maiúsculas em excesso
  • Grande bloco de texto em negrito ou itálico
  • Texto pequeno demais
  • Texto sem link está sublinhado
  • Texto justificado
  • Subscrito ou sobrescrito usado de forma incorreta como formatação
  • Lista falsa usando caracteres ou símbolos
  • Item de lista fora de uma lista

Contraste de cores

  • Texto tem contraste insuficiente
  • Contraste do texto requer verificação manual
  • Ícone ou elemento gráfico tem contraste insuficiente
  • Contraste de ícone ou elemento gráfico requer verificação manual
  • Texto de campo de entrada tem contraste insuficiente
  • Texto de placeholder tem contraste insuficiente

Tabelas

  • Tabela sem linha ou coluna de cabeçalho
  • Célula de cabeçalho da tabela está vazia
  • Título de conteúdo usado dentro de uma tabela

Formulários & elementos interativos

  • Botão sem rótulo acessível
  • Botão tem rótulo ARIA inválido
  • Rótulo do botão inclui a palavra "botão"
  • Rótulo visível não corresponde ao nome acessível
  • Campo sem rótulo associado
  • Campo usa apenas um rótulo invisível
  • Campo usa apenas um placeholder como rótulo
  • Botão de redefinição pode causar perda acidental de dados
  • Elemento oculto de leitores de tela ainda recebe foco pelo teclado
  • tabindex positivo quebra a ordem de leitura e navegação

Metadados da página

  • Título da página ausente
  • Idioma da página não declarado
  • Viewport impede o redimensionamento do texto
  • Página atualiza automaticamente

Funcionalidades da CSA

O Editoria11y promove a acessibilidade de uma forma única. Suas ferramentas são altamente eficazes para ajudar autores não técnicos a preparar conteúdo que possa ser aproveitado igualmente por usuários da Web com deficiência. Consideramos isso um bem público, por isso o Editoria11y sempre será gratuito para usar.

O Editoria11y, no entanto, não é gratuito para desenvolver ou oferecer suporte.

A CSA preenche essa lacuna: os membros do projeto financiam o desenvolvimento da biblioteca Editoria11y, seus plugins de CMS e o conjunto CSA: um conjunto de ferramentas de código aberto para garantia de qualidade, em rápido crescimento, que oferece funcionalidades similares às de produtos comerciais, com um custo muito mais acessível. O conjunto CSA para Drupal está atualmente gratuito em versão beta, e chegará ao WordPress no terceiro trimestre de 2026.

Testes & Ferramentas

  • Testes para desenvolvedores, legibilidade e contraste.
  • Configuração dividida por função para desenvolvedores e criadores de conteúdo.
  • Construtor de testes personalizados.
  • Dispensa de alertas em todo o site com um clique.
  • Ferramentas de manutenção de painel e rastreadores.

Benefícios para a comunidade

  • Suporte prioritário.

  • Assistência direta na configuração (conforme o nível).
  • Crédito público de apoiador (conforme o nível/mediante solicitação; termos se aplicam).
