Funktionen

Die Kerntests von Editoria11y erkennen mehr als 50 häufige Barrierefreiheitsprobleme bei Inhalten: Entscheidungen, die die Lesbarkeit beeinträchtigen, Fehler, die Screenreader weniger effektiv machen, und Probleme, die die SEO verschlechtern.

Add-ons bieten ~40 weitere Tests für Entwickler und Designer, benutzerdefinierte Tests und verschiedene seitenweite Qualitätssicherungswerkzeuge.

Die Inhaltsprüfungen konzentrieren sich auf Probleme, die nach der Übergabe der Website an Content-Autoren auftreten. Websites müssen vor dem Launch mit manuellen Prüfwerkzeugen, Tastaturen und Screenreadern getestet werden — aber selbst die barrierefreieste Website neigt dazu, sich mit der Zeit zu verschlechtern, wenn Bearbeitungen neue Probleme einbringen.

Was Editoria11y unter den Barrierefreiheitswerkzeugen auszeichnet, ist dass seine Tests live im Browser ausgeführt werden, während Autoren ihre Inhalte bearbeiten und in der Vorschau ansehen. Autoren reagieren eher auf unmittelbares, direktes Feedback, bemerken Warnungen und nehmen Korrekturen vor, als dass sie daran denken, ein Dashboard zu besuchen oder eine manuelle Prüfung durchzuführen.

Die Autorenerfahrung

Autoren sehen die Editoria11y-Symbolleiste beim Bearbeiten von Inhalten. Wenn Probleme erkannt werden, wird sie gelb oder rot und zeigt eine Problemanzahl an. Ein Klick auf die Anzahl scrollt zum ersten Hinweis und öffnet seinen Tooltip.

Hinweise erscheinen inline mit einer einfachsprachigen Fehlerbeschreibung, Korrekturvorschlägen und einer Erklärung, warum das Problem wichtig ist.

Die Symbolleiste enthält zusätzliche Werkzeuge für manuelle Prüfungen, darunter Visualisierer für Überschriftenstruktur und Alternativtexte sowie eine Lesbarkeitsauswertung.

Steuerungsmöglichkeiten für Administratoren

Jeder sichtbare Aspekt des Prüfers kann angepasst werden. Parameter und Ereignisse ermöglichen das Entfernen von Tests, das Umschreiben von Hinweisen, das Einfügen benutzerdefinierter Tests und das Festlegen benutzerdefinierter Themes.

Plugins für Drupal und WordPress fügen weitere Funktionen hinzu, wie z. B. synchronisierte Abweisungen, seitenweite Dashboards und CSV-Berichtsexporte.

Tests

Die Tests von Editoria11y zielen darauf ab, das Web-Erlebnis für alle Nutzer zu verbessern — gemessen an menschlichen Maßstäben, nicht nur an technischen Standards. Während wichtige Tests dazu beitragen sollen, die Web Content Accessibility Guidelines (WCAG 2.2 AA) zu erfüllen, werden andere von der Arbeit der Cognitive and Learning Disabilities Accessibility Task Force sowie allgemeinen Best Practices für Benutzerfreundlichkeit und Design beeinflusst.

Einzelne Tests können von Website-Administratoren deaktiviert werden.

CSA-Funktionen

Editoria11y fördert Barrierefreiheit auf einzigartige Weise. Seine Werkzeuge sind sehr effektiv dabei, nicht-technischen Autoren zu helfen, Inhalte zu erstellen, die von Webnutzern mit Behinderungen gleichermaßen genutzt werden können. Wir betrachten dies als ein öffentliches Gut, daher wird Editoria11y immer kostenlos nutzbar sein.

Editoria11y ist jedoch nicht kostenlos zu entwickeln oder zu unterstützen.

Das CSA schließt diese Lücke: Projektmitglieder finanzieren die Entwicklung der Editoria11y-Bibliothek, seiner CMS-Plugins und der CSA-Suite: eine schnell wachsende Sammlung von Open-Source-Qualitätssicherungswerkzeugen, die ähnliche Funktionen wie kommerzielle Produkte zu einem deutlich günstigeren Preis bieten. Die CSA-Suite für Drupal ist derzeit kostenlos in der Beta und wird für WordPress im 3. Quartal 2026 verfügbar sein.

Tests & Werkzeuge

  • Entwickler-, Lesbarkeits- und Kontrasttests.
  • Rollenbasierte getrennte Konfiguration für Entwickler und Content-Ersteller.
  • Benutzerdefinierte Test-Builder.
  • Seitenweite Abweisungen per Klick
  • Dashboard-Verwaltungswerkzeuge und Crawler

Community-Vorteile

  • Prioritätssupport.
  • Gewichteter Einfluss auf die Projekt-Roadmap.
  • Automatischer Zugang zu neuen Funktionen auf der Roadmap.
  • Direkte Einrichtungshilfe (je nach Stufe).
  • Öffentliche Unterstützernennungen (je nach Stufe/auf Anfrage; Bedingungen gelten).

Contribute-what-you-can-Lizenzierung

Nach der Beta wird die CSA-Suite über ein Contribute-what-you-can-Modell verfügbar gemacht. Editoria11y Core — sowohl die Bibliothek als auch die Plugins — bleibt kostenlos.
