Características

Las pruebas principales de Editoria11y detectan más de 50 problemas comunes de accesibilidad en el contenido: decisiones que perjudican la legibilidad, errores que dificultan el uso de lectores de pantalla y fallos que reducen el SEO.

Los complementos ofrecen ~40 pruebas adicionales para desarrolladores y diseñadores, pruebas personalizadas y diversas herramientas de control de calidad para todo el sitio.

Las pruebas de contenido se centran en problemas que aparecen después de que los sitios se entregan a los autores de contenido. Los sitios deben probarse con herramientas de auditoría manual, teclados y lectores de pantalla antes del lanzamiento, pero incluso el sitio más accesible el primer día tiende a deteriorarse a medida que las ediciones introducen nuevos problemas.

Lo que distingue a Editoria11y entre las herramientas de accesibilidad es que sus pruebas se ejecutan en vivo, en el navegador, mientras los autores editan y previsualizan su trabajo. Los autores tienen más probabilidades de reaccionar ante comentarios inmediatos y en línea, notar alertas y hacer correcciones, que a recordar visitar un panel o ejecutar una verificación manual.

La experiencia del autor

Los autores ven la barra de herramientas de Editoria11y mientras editan el contenido. Cuando se detectan problemas, se vuelve amarilla o roja y muestra el número de incidencias. Al hacer clic en el recuento, se desplaza hasta la primera alerta y se abre su información emergente.

Los consejos aparecen en línea, con una descripción en lenguaje sencillo del error, sugerencias de corrección y notas sobre por qué es importante el problema.

La barra de herramientas contiene herramientas adicionales para verificaciones manuales, incluidos visualizadores de esquema de encabezados, texto alternativo y puntuación de legibilidad.

Controles del administrador del sitio

Cada aspecto visible del verificador puede personalizarse. Los parámetros y eventos permiten eliminar pruebas, reescribir consejos, insertar pruebas personalizadas y establecer temas personalizados.

Los complementos para Drupal y WordPress añaden funciones adicionales, como descartes sincronizados, paneles de todo el sitio y exportaciones de informes en CSV.

Pruebas

Las pruebas de Editoria11y se centran en mejorar la experiencia de la Web para todos los usuarios, medida por personas, no solo por estándares técnicos. Si bien las pruebas clave pretenden ayudar a cumplir las Pautas de Accesibilidad al Contenido en la Web (WCAG 2.2 AA), otras se inspiran en el trabajo del Grupo de Trabajo de Accesibilidad para Discapacidades Cognitivas y de Aprendizaje, así como en las mejores prácticas generales de usabilidad y diseño.

Los administradores del sitio pueden desactivar las pruebas individualmente.

Texto alternativo de imágenes

  • Imagen sin atributo alt
  • El texto alternativo es un nombre de archivo
  • El texto alternativo es un marcador de posición ("TBD")
  • El texto alternativo contiene símbolos impronunciables
  • El texto alternativo contiene palabras redundantes ("imagen de…")
  • El texto alternativo es demasiado largo
  • La imagen decorativa puede tener valor
  • La imagen de carrusel está marcada como decorativa
  • La imagen con pie de foto no tiene texto alternativo
  • El texto alternativo duplica el pie de foto

Imágenes enlazadas

  • La imagen enlazada no tiene texto alternativo
  • El texto alternativo de la imagen enlazada es una URL o un nombre de archivo
  • El texto alternativo de la imagen enlazada es un marcador de posición
  • El texto alternativo de la imagen enlazada contiene símbolos impronunciables
  • El texto alternativo de la imagen enlazada describe la imagen, no el enlace
  • El texto alternativo de la imagen enlazada contiene palabras redundantes
  • El texto alternativo de la imagen enlazada es demasiado largo
  • La imagen en un enlace con texto no tiene atributo alt

Contenido multimedia incrustado

  • El video necesita subtítulos
  • El audio necesita una transcripción
  • La visualización de datos necesita una alternativa accesible
  • El marco (frame) no tiene título
  • El marco está excluido de la navegación por teclado
  • El contenido del marco necesita una verificación manual de accesibilidad
  • El documento enlazado puede no ser accesible para lectores de pantalla
  • El PDF carece de una alternativa accesible
  • Componentes de diseño interactivos anidados

Enlaces significativos

  • Enlace vacío
  • El texto del enlace es una URL
  • El texto del enlace es solo un número DOI
  • El texto del enlace dice "haz clic aquí"
  • El texto del enlace solo contiene palabras genéricas ("leer más")
  • Diferentes enlaces comparten el mismo texto
  • El icono o imagen enlazados no tienen alternativa de texto
  • El enlace abre una nueva pestaña sin advertencia
  • El enlace apunta a un archivo sin advertencia
  • El texto del enlace son solo símbolos o emojis
  • Texto de enlace significativo oculto para usuarios con visión
  • Información emergente de enlace redundante
  • Enlace de anclaje interno roto
  • Atributo ID duplicado
  • El enlace puede apuntar a un entorno de desarrollo

Encabezados

  • La página no tiene Encabezado 1
  • El primer encabezado es un subencabezado
  • El encabezado salta un nivel
  • El encabezado está vacío
  • La imagen usada como encabezado necesita texto alternativo
  • El encabezado es muy largo
  • Un párrafo en negrita podría ser un encabezado
  • Una cita corta podría ser un encabezado

Legibilidad del texto

  • Texto excesivo en mayúsculas
  • Bloque grande de texto en negrita o cursiva
  • El texto es demasiado pequeño
  • El texto sin enlace está subrayado
  • El texto está justificado
  • Subíndice o superíndice mal usado como formato
  • Lista falsa con caracteres o símbolos
  • Elemento de lista fuera de una lista

Contraste de color

  • El texto tiene contraste insuficiente
  • El contraste del texto requiere verificación manual
  • El icono o gráfico tiene contraste insuficiente
  • El contraste del icono o gráfico requiere verificación manual
  • El texto de entrada tiene contraste insuficiente
  • El texto de marcador de posición tiene contraste insuficiente

Tablas

  • La tabla no tiene fila o columna de encabezado
  • La celda de encabezado de tabla está vacía
  • Encabezado de contenido usado dentro de una tabla

Formularios y elementos interactivos

  • El botón no tiene etiqueta accesible
  • El botón tiene una etiqueta ARIA no válida
  • La etiqueta del botón incluye la palabra "botón"
  • La etiqueta visible no coincide con el nombre accesible
  • La entrada no tiene etiqueta asociada
  • La entrada usa solo una etiqueta invisible
  • La entrada usa solo un marcador de posición como etiqueta
  • El botón de reinicio puede causar pérdida accidental de datos
  • Elemento oculto para lectores de pantalla pero con foco de teclado activo
  • El tabindex positivo rompe el orden de lectura y tabulación

Metadatos de página

  • Título de página no declarado
  • Idioma de página no declarado
  • La ventana gráfica impide el escalado del texto
  • La página se actualiza automáticamente

Funciones CSA

Editoria11y promueve la accesibilidad de una forma única. Sus herramientas son muy eficaces para ayudar a los autores no técnicos a preparar contenido que puedan disfrutar por igual los usuarios de la Web con discapacidad. Consideramos esto un bien público, por lo que Editoria11y siempre será gratuito de usar.

Sin embargo, desarrollar y mantener Editoria11y no es gratis.

El CSA cubre la diferencia: los miembros del proyecto financian el desarrollo de la biblioteca Editoria11y, sus complementos para CMS y el paquete CSA: un conjunto de herramientas de control de calidad de código abierto en rápida expansión que ofrecen funcionalidad similar a la de los productos comerciales, a un precio mucho menor. El paquete CSA para Drupal está actualmente en versión beta gratuita y llegará a WordPress en el tercer trimestre de 2026.

Pruebas y herramientas

  • Pruebas de desarrollador, legibilidad y contraste.
  • Configuración dividida por rol para desarrolladores y creadores de contenido.
  • Constructores de pruebas personalizadas.
  • Descartes globales del sitio con un clic.
  • Herramientas de mantenimiento de paneles y rastreadores.

Beneficios para la comunidad

  • Soporte prioritario.
  • Influencia ponderada en la hoja de ruta del proyecto.
  • Acceso automático a las nuevas funciones de la hoja de ruta.
  • Asistencia directa para la configuración (según el nivel).
  • Crédito público como patrocinador (según nivel/por solicitud; se aplican condiciones).

Licencia de contribuye lo que puedas
